Full MERN Stack Architecture Overview

To transform your portfolio from a static React client into a dynamic full-stack application, we introduce an Express API Server running on Node.js and a MongoDB Database. This architecture will handle secure authentication, save form submissions, and serve dynamic data for projects, skills, and certifications.

Data Flow Cycle

  1. Guest Submits Form: Guest inputs their contact packet on the portfolio frontend. An HTTP POST request transmits the packet to the backend server.
  2. Backend Database Storing: Express server validates the incoming data schema, and inserts a document into the MongoDB Atlas database.
  3. Admin Logs In: Admin navigates to /admin, inputs credentials, and requests authorization. The backend signs a JWT (JSON Web Token).
  4. Secure Management Operations: Using the secure token, the Admin panel queries MongoDB to list incoming contact messages, update skills, or publish new projects.

MongoDB Database Schema Models

We will use Mongoose in Node.js to define structured schemas for our database collections. Below are the models designed to host your site's data.

1. Message Collection (Inbox Logs)

const messageSchema = new mongoose.Schema({
  name: { type: String, required: true },
  email: { type: String, required: true },
  phone: { type: String },
  message: { type: String, required: true },
  isRead: { type: Boolean, default: false },
  createdAt: { type: Date, default: Date.now }
});

2. Project Collection

const projectSchema = new mongoose.Schema({
  title: { type: String, required: true },
  desc: { type: String, required: true },
  category: { type: String, enum: ['frontend', 'fullstack'], required: true },
  tags: [{ type: String }],
  github: { type: String },
  live: { type: String },
  image: { type: String }, // URL pointer
  createdAt: { type: Date, default: Date.now }
});

3. Certification Collection

const certificationSchema = new mongoose.Schema({
  title: { type: String, required: true },
  issuer: { type: String, required: true },
  date: { type: String, required: true },
  credId: { type: String },
  image: { type: String } // URL preview link
});

API Routing Gateways (Express.js)

The Express server will expose these API routes. Protected endpoints require a valid JWT token passed in the Authorization: Bearer request header.

Route Endpoint HTTP Method Protection Functional Description
/api/auth/login POST Public Verifies username/password, generates JWT authorization token.
/api/messages POST Public Submits a new guest contact message packet to MongoDB database.
/api/messages GET Admin Protected Fetches all received inbox form packets.
/api/messages/:id DELETE Admin Protected Deletes an inbox message packet by its Database ID.
/api/projects GET Public Retrieves all projects stored in database to render dynamically in UI.
/api/projects POST Admin Protected Creates a new project record card inside MongoDB.
/api/projects/:id PUT / DELETE Admin Protected Modifies or removes a project record from database.

Current Frontend Audit & Bug Analysis

I performed a detailed analysis of the existing React codebase. Below is the list of identified bugs, performance issues, and optimization points, alongside their corresponding fixes.

File Path Bug / Performance Issue Status Resolution Details / Fix Implemented
CustomCursor.jsx Uses left and top CSS properties directly on mousemove. This triggers frequent layout reflows on every single frame, causing stutters and lag. Resolved Refactored the inline styling and layout properties to use hardware-accelerated transform: translate3d(x, y, 0), eliminating layout reflow recalculations.
Navbar.jsx When the responsive mobile menu overlay is opened on mobile, the background page body is still scrollable. Resolved Added a useEffect observer inside the component that dynamically locks the body scroll overflow (overflow = 'hidden') whenever the mobile navigation overlay is open.
Contact.jsx Missing standard email validation regex and phone number check. Clicking submit allows empty or corrupted email submissions. Resolved Implemented strict regex format validation (/^[^\s@]+@[^\s@]+\.[^\s@]+$/) inside the submit handler, displaying a glowing neon-pink cyber error prompt beneath the input.
SPA Routing Config As a Single Page App (SPA), reloading the browser directly at a URL path like /admin returns a server 404 Not Found unless fallback routing is configured. Resolved Created Netlify's public/_redirects and vercel.json rewrites routing all sub-path queries back to index.html to prevent route-refresh failures.
