Maintenance Scheduling Sequence

Detailing the operations that trigger, maintain, and clear public site maintenance modes.

Scheduling Operations Flow Protocol Execution
01

Admin Configuration Settings

The admin determines whether to toggle the maintenance mode immediately using a **Preset Duration** (Manual, 5m, 15m, 30m, 1h, 2h) or scheduling it for a specific target time using the **Custom Date & Time** picker.

02

REST Dispatch to Database

The admin client sends a payload to `POST /api/analytics/maintenance`. If scheduling, `maintenanceEnd` is stored in the database. If relative, `maintenanceEnd` is computed as `Date.now() + durationMinutes * 60000`.

03

Public Client Routing Intercept

Visits to `/` query `/api/analytics/maintenance-status`. If `maintenanceMode` is true, the router intercepts the rendering tree, mounting `` and passing `maintenanceEnd` to initiate the ticking countdown.

04

Timer Evaluation & Automatic Reset

On visitor query checks or admin status updates, the server checks if `new Date() > stats.maintenanceEnd`. Once this evaluates to true, `maintenanceMode` resolves to `false`, clearing the lock and bringing the site online instantly.
